umgr

Provider Authoring Guide

Build platform adapters that validate desired account state, import current users, plan drift changes, and apply reconciliation safely.

Provider Responsibilities

A provider must implement four methods:

  • validate(resource:)
  • current(resource:)
  • plan(desired:, current:)
  • apply(changeset:)

See implementation contract in lib/umgr/provider.rb.

Validation Contract

  • Preferred: raise Umgr::Errors::ValidationError.
  • Also supported: return { ok: false } (symbol or string keys); umgr promotes it to a validation error.

Current State Contract (Import)

Supported current(resource:) success response shapes:

  • Recommended: { ok: true, imported_accounts: [resource_hash, ...] }
  • Also supported: { ok: true, resource: resource_hash }
  • Also supported: { ok: true, account: account_hash } where account is a hash mapped into attributes

Failure shape: { ok: false, error: "..." } (symbol or string keys).

Contract Enforcement

Registration is validated by Umgr::ProviderContract:

  • Provider object must respond to required methods.
  • Methods must be concrete (not inherited abstract implementations).
  • Failures raise Umgr::Errors::ProviderContractError.

Registry and Naming

  • Register via register(name, provider_instance).
  • Name is normalized to symbol (for example, "echo" -> :echo).
  • Empty names raise Umgr::Errors::ProviderContractError.

Resource Shape

Required resource keys:

  • provider
  • type
  • name

Canonical identity is provider.type.name.

Provider code should read deep-symbolized keys as symbols.

Runtime Flow

  1. Runner resolves config path (explicit --config or auto-discovery).
  2. Config is schema-validated.
  3. Unknown providers are rejected early.
  4. Action execution uses providers from the registry.

Minimal Provider Example

# lib/umgr/providers/acme_provider.rb
# frozen_string_literal: true

module Umgr
  module Providers
    class AcmeProvider < Provider
      def validate(resource:)
        { ok: true, provider: "acme", resource: resource }
      end

      def current(resource:)
        account = { email: resource.dig(:attributes, :email) }
        { ok: true, provider: "acme", account: account }
      end

      def plan(desired:, current:)
        status = desired == current ? "no_change" : "update"
        { ok: true, provider: "acme", status: status, desired: desired, current: current }
      end

      def apply(changeset:)
        { ok: true, provider: "acme", status: "applied", changeset: changeset }
      end
    end
  end
end

Registering a Provider

require "umgr"
require "umgr/providers/acme_provider"

registry = Umgr::ProviderRegistry.new
registry.register(:acme, Umgr::Providers::AcmeProvider.new)

runner = Umgr::Runner.new(provider_registry: registry)
result = runner.dispatch(:validate, config: "users.yml")

Error Behavior

  • Umgr::Errors::ValidationError for invalid input/config.
  • Umgr::Errors::ProviderContractError for bad provider contracts.
  • Umgr::Errors::InternalError for unexpected runtime issues.

Testing Guidance

  • Library coverage with RSpec for provider method branches.
  • CLI coverage with Aruba for command/exit behavior.
bundle exec rubocop
bundle exec rspec --exclude-pattern "spec/cli/**/*_spec.rb"
bundle exec rspec spec/cli
